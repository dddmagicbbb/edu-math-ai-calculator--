# 애플리케이션 실행 및 종료 가이드

본 문서는 '학습 밀착형 AI 공학계산기'를 로컬 환경에서 실행하고 안전하게 종료하는 방법을 안내합니다.

## 1. 실행 방법 (Run)

### 1.1 환경 변수 설정 (`.env`)
프로젝트 루트 폴더(`d:\Dev\projects\시작`)에 `.env` 파일이 있어야 합니다.
- `.env.example` 파일을 복사하여 `.env`로 이름을 변경합니다.
- 파일 내용 중 `GROQ_API_KEY` 부분에 자신의 API 키를 입력합니다.
  ```env
  GROQ_API_KEY=your_actual_api_key_here
  ```

### 1.2 서버 실행 (PowerShell 또는 CMD)
가상 환경(venv)을 사용하는 경우, 반드시 활성화 상태에서 진행하세요.

1.  **가상 환경 활성화** (사용하는 경우)
    ```powershell
    .\.venv\Scripts\Activate.ps1
    ```
2.  **의존성 패키지 설치** (가상 환경 활성화 후 최초 1회 필수)
    ```powershell
    pip install -r requirements.txt
    ```
3.  **Flask 서버 가동**
    ```powershell
    python api/index.py
    ```

### 1.3 웹 접속
서버가 정상적으로 실행되면 브라우저를 열고 다음 주소로 접속합니다.
- **주소**: [http://127.0.0.1:5000](http://127.0.0.1:5000)

---

## 2. 종료 방법 (Shutdown)

### 2.1 터미널에서 종료 (추천)
서버가 실행 중인 터미널 창에서 다음 단축키를 입력합니다.
- **단축키**: `Ctrl + C` (Control 키와 C 키를 동시에 누름)
- 입력 후 서버 로그가 멈추고 명령 입력 상태로 돌아오면 성공적으로 종료된 것입니다.

### 2.2 터미널 창 닫기
단축키가 작동하지 않거나 급한 경우, 서버가 실행 중인 터미널(PowerShell/CMD) 창을 직접 닫아도 프로세스가 종료됩니다.

---

## 3. 문제 해결 (Troubleshooting)
- **포트 충돌**: `Address already in use` 오류가 발생하면, 포트 5000을 사용하는 다른 프로세스를 종료하거나 `api/index.py` 파일 하단의 `port=5000`을 다른 번호로 변경하세요.
- **ModuleNotFoundError**: `flask` 등의 모듈을 찾을 수 없다는 오류가 나오면, 가상 환경을 활성화한 상태에서 `pip install -r requirements.txt`를 다시 실행해 보세요.
